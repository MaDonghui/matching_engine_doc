Detailed Description

An "Central Limit Order Book", contain all order from both side and metadata
Internal correctness is guaranteed via guarded modifying methods
Meaning Orders are in correct storage position and priority and metadata is updated when necessary

◆ detach()

Order * TradeDS::Book::detach ( uint64_t  order_id)

Remove an existing TradeDS::Order from it's TradeDS::Limit without destructing it
TraderDS::Order's chaining information will be reset
Book's metadata will be updated as the order is no longer in the book

time-complexity
O(1) on average
O(L) on worst case; where L is the number of Limit on Order's Side. this happens when Order is the only order in its TradeDS::Limit, and all Limit are empty
updating Book's metadata will resulting a full Limit iteration over one Side
